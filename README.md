# Proyecto: Modelos Anatómicos - Segmento de Columna 

Bienvenidas al equipo de desarrollo! En este repositorio trabajaremos en la creación de un **modelo simplificado de un segmento de columna vertebral**. Este modelo servirá como material didáctico para exponer las vértebras y sus estructuras internas.

---

## Estructura del Repositorio

Para que el trabajo de ambas esté organizado, utilizaremos carpetas por semana. Cada una de ustedes deberá subir su propio reporte o trabajar en conjunto en uno solo, según se coordinen.

```text
ModelosAnatomicos/
└── Columna/
    ├── documentación/       <-- (informe, Fotos de prototipos, planos, capturas, etc.)
    ├── modelos/             <-- (STL, DICOM, etc)
    └── papers/              <-- (pdfs y documentos de interés)

```

---

## Guía de Trabajo (Flujo de Colaboración)

Como este es un repositorio compartido, sigan estos pasos para mantener el orden:

### 1. Cargar Archivos y Fotos
Pueden subir archivos (como fotos del modelo físico o diseños) usando el botón **"Add file"** -> **"Upload files"**. 
*   **Importante:** Asegúrense de guardar las imágenes en la carpeta correspondiente.

### 2. El "COMMIT" (ojo!)
GitHub no guarda cambios automáticamente. Para que sus archivos se publiquen y yo pueda verlos, deben realizar un **Commit**:
1. Al subir o editar un archivo, bajen hasta el cuadro **"Commit changes"**.
2. Escriban una descripción corta (ejemplo: *"Subiendo fotos de la vértebra L1"*).
3. Hagan clic en el botón verde **"Commit changes"**. 
4. **Si no hacen clic en el botón verde, el trabajo no se guarda.**

### 3. Notificación de Avance (Issues)
Cada vez que completen una tarea o un avance, deben avisarme así:
1. Vayan a la pestaña **"Issues"** arriba.
2. Clic en **"New Issue"**.
3. **Título:** `Avance del día [Fecha] - [Sus Nombres]`.
4. **Descripción:** Resuman qué hicieron hoy y **etiquétenme** usando: `@oscampo`. También pueden etiquetar a su compañera si necesitan que ella revise algo.
5. Clic en **"Submit new issue"**.

---

## 💡 Reglas de Oro para el trabajo en equipo
*   **No borren archivos de su compañera** sin consultarlo antes.
*   **Nombres de archivos claros:** Si suben una foto, no la dejen como `IMG_452.jpg`. Cámbienle el nombre a algo como `Prototipo_Cervical_V1.jpg`.
*   **Comuníquense en los Issues:** Si tienen una duda técnica, pueden abrir un Issue y etiquetarnos a todos para resolverlo ahí mismo.

## Tips de Formato (Markdown)
Para que sus reportes se vean profesionales:
*   Usen `#` para Títulos y `##` para Subtítulos.
*   Para poner una imagen en el reporte usen: `![texto](ruta/a/la/imagen.jpg)`
*   Usen `**negritas**` para resaltar puntos clave.
